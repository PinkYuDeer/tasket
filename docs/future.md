# future

## 原则

此文只载发布后或非首版必需之远期功能。凡会影响当前发布稳定性的事项，应移回 `docs/plan.md` 并配测试与验收。

## GUI 完整体验

- 加加载中、失败重试、请求中状态、版本冲突提示、无权限提示。
- 删除操作加确认面板，防误触。
- 整理子任务树：懒加载、递归展开、排序缓存失效。
- 标签 UI：筛选、颜色显示、按团队/私有/公开分类管理。
- 全部硬编码英文迁至 `zh_CN.lang` / `en_US.lang`。

## 同步与网络增强

- `task_action`、`team_action` 增 `requestId` 与客户端待确认状态。
- 任务同步支持分页、按 `update_time` 增量、按团队/标签过滤。
- 服务端定时轮询 BQ/GTNHLib-linked team，刷新外部成员快照。
- 网络包增加协议版本协商与兼容降级。

## 团队与外部集成

- GTNHLib `ITeamData` 承载实际 tasket 团队统计、团队设置或迁移游标。
- BetterQuesting 任务书进度适配：设计 `TaskType.QUEST_FINISH` 或独立关联表。
- 若 BQ 全面迁入 GTNHLib Team，收敛 provider，仅保历史 BQ party 迁移入口。
- 公开轻量 API，供其他 Mod 查询或创建待办、标签、团队任务。

## 协作功能

- 评论、关注、点赞、提醒、审批、进度更新。
- 通知中心：团队邀请、任务分配、状态变更、提醒、系统警告。
- 周期任务：先支持简单 interval，再扩展 RRULE。
- 任务分配与多人执行记录：负责人、协作者、工作时长、完成证明。

## 视图与可视化

- 标签看板、团队看板、按成员筛选。
- 时间轴、甘特图、任务依赖图。
- 思维导图式任务树。
- 只读统计页：任务完成率、团队贡献、标签分布。

## 奖励与长期玩法

- 积分、等级、贡献值、团队统计。
- 可配置奖励规则与成就任务。
- 与 GTNH 任务书、成就、阶段进度的非破坏性联动。

## 数据与架构长期治理

- 统一 ID 边界：评估 `Task.id` 从 `String` 收敛为 `UUID` 的迁移成本。
- DAO 泛型基类或模板，减少重复代码。
- 复杂迁移框架：表重建、回滚、迁移前备份。
- 压测：大量任务、标签、团队成员、高频同步。
